## Введение

Данный документ описывает план тестирования приложения ["BookTracker"](https://github.com/JChanK/Book-collection). 
Документ предназначен для людей, выполняющих тестирование данного проекта.
Цель тестирования - проверка соответствия реального поведения программы проекта и ее ожидаемого поведения, которое описано в требованиях и диаграммах проекта.

<a name="items"/>

## Объект тестирования

В качестве объектов тестирования можно выделить следующие функциональные требования:

* Вход пользователя в приложение;
* Регистрация нового пользователя;
* Поиск книг и информации о них;
* Управление книжной коллекцией;
* Просмотр детальной информации о книгах и авторах;
* Работа с рецензиями;
* Выход пользователя из учётной записи.

<a name="quality"/>

## Атрибуты качества

1. Функциональность:
    - функциональная полнота: приложение должно выполнять все заявленные функции;
    - функциональная корректность: приложение должно выполнять все заявленные функции корректно.
2. Удобство использования:
    - интуитивно понятный интерфейс;
    - минималистичный дизайн;
    - адаптивность для различных разрешений экрана.
3. Производительность:
    - эффективная работа с базой данных;
    - оптимальное использование API внешних сервисов.
    
<a name="risk"/>

## Риски

В данном случае к рискам можно отнести:
* Отсутствие интернет-соединения, что ограничит доступ к внешним API книжных сервисов;
* Проблемы с доступностью внешних API;
* Ошибки при работе с базой данных;
* Некорректное отображение интерфейса на различных разрешениях экрана.

<a name="features"/>

## Аспекты тестирования

В ходе тестирования планируется проверить реализацию основных функций приложения. Аспекты, подвергаемые тестированию: 

### Авторизация и регистрация
Этот вариант использования необходимо протестировать на:
* Вход в приложение под существующим пользователем;
* Выход из учетной записи;
* Валидация данных при регистрации;
* Регистрация нового пользователя.

### Управление коллекцией
Этот вариант использования необходимо протестировать на:
* Добавление книг в коллекцию; 
* Удаление книг из коллекции;
* Изменение статуса книг в коллекции;
* Отображение персональной коллекции; 

### Поиск и информация
Этот вариант использования необходимо протестировать на:
* Поиск книг через каталог;  
* Поиск авторов;
* Просмотр детальной информации о книгах;  
* Просмотр информации об авторах.

### Рецензии
Этот вариант использования необходимо протестировать на:
* Добавление рецензий к книгам;
* Редактирование рецензий;
* Отображение рецензий на страницах книг;
* Модерация рецензий (если предусмотрено).

### Интерфейс пользователя
Этот вариант использования необходимо протестировать на:
* Корректность отображения всех элементов интерфейса;
* Работоспособность навигации между экранами;
* Адаптивность под разные разрешения экрана.

### Функциональность для разных типов пользователей
Этот вариант использования необходимо протестировать на:
* Ограничения функционала для гостевых пользователей;
* Полный функционал для зарегистрированных пользователей;

## Нефункциональные требования:
* Производительность при работе с базой данных;
* Безопасность хранения пользовательских данных;
* Удобство использования интерфейса.

## Выводы
Данный тестовый план позволяет протестировать основной функционал приложения "BookTracker". Успешное прохождение всех тестов не гарантирует полной работоспособности на всех версиях платформ и архитектурах, однако позволяет полагать, что данное программное обеспечение работает корректно. Особое внимание следует уделить тестированию интеграции с внешними API и работе приложения в условиях отсутствия интернет-соединения.
